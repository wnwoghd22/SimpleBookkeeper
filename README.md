# 복식부기 가계부 앱 - Claude Code 개발 가이드

## 프로젝트 개요
이 프로젝트는 복식부기 원리를 적용한 간단한 가계부 웹 애플리케이션입니다.
로컬 스토리지를 사용하여 데이터를 저장하고, XLSX 형식으로 내보내기/불러오기가 가능합니다.

## 파일 구조
```
double-entry-bookkeeping/
├── index.html          # 메인 HTML 파일

└── README.md          # 이 파일
```

## 주요 기능

### 1. CRUD 기능
- **Create**: 거래 추가 (날짜, 적요, 차변/대변 계정 및 금액)
- **Read**: 거래 목록 조회 및 필터링
- **Update**: 거래 수정
- **Delete**: 거래 삭제

### 2. 복식부기 원리
- 모든 거래는 차변(Debit)과 대변(Credit)으로 구성
- 차변과 대변의 금액은 항상 동일해야 함
- 계정은 자산, 비용, 수익으로 분류

### 3. 데이터 저장
- **로컬 스토리지**: 브라우저에 자동 저장
- **엑셀 내보내기**: XLSX 형식으로 저장
- **엑셀 불러오기**: XLSX 파일에서 데이터 가져오기

### 4. 요약 기능
- 계정별 잔액 실시간 계산
- 시각적인 카드 형태로 표시

### 기능 추가 요청 예시

#### 1. 월별 통계 기능 추가
```
이 가계부 앱에 월별 수입/지출 통계를 그래프로 보여주는 기능을 추가해줘.
```

#### 2. 예산 관리 기능 추가
```
각 비용 계정(식비, 교통비 등)에 월별 예산을 설정하고, 
현재 지출이 예산의 몇 %인지 보여주는 기능을 추가해줘.
```

#### 3. 반복 거래 등록 기능
```
매월 반복되는 거래(급여, 월세 등)를 미리 등록해두고,
한 번의 클릭으로 해당 월에 자동으로 추가되는 기능을 만들어줘.
```

#### 4. CSV 지원 추가
```
XLSX 외에 CSV 형식으로도 내보내기/불러오기가 가능하도록 해줘.
```

#### 5. 다크모드 추가
```
다크모드를 추가하고, 사용자가 토글로 전환할 수 있게 해줘.
localStorage에 설정을 저장해서 다음 방문시에도 유지되도록 해줘.
```

#### 6. 검색 기능 강화
```
거래 내역에서 적요를 검색할 수 있는 기능을 추가하고,
금액 범위로도 필터링할 수 있게 해줘.
```

## XLSX 파일 형식

### 거래내역 시트
| 날짜 | 적요 | 차변계정 | 차변금액 | 대변계정 | 대변금액 |
|------|------|----------|----------|----------|----------|
| 2025-01-01 | 급여 입금 | 은행예금 | 3000000 | 급여 | 3000000 |
| 2025-01-02 | 점심 식사 | 식비 | 12000 | 현금 | 12000 |

### 계정별잔액 시트
| 계정 | 분류 | 잔액 |
|------|------|------|
| 은행예금 | 자산 | 2988000 |
| 현금 | 자산 | -12000 |
| 식비 | 비용 | 12000 |
| 급여 | 수익 | 3000000 |

## 테스트 데이터 생성 방법

```javascript
// 브라우저 콘솔에서 실행
const testTransactions = [
    {
        id: Date.now().toString(),
        date: '2025-01-01',
        description: '1월 급여',
        debitAccount: '은행예금',
        debitAmount: 3000000,
        creditAccount: '급여',
        creditAmount: 3000000
    },
    {
        id: (Date.now() + 1).toString(),
        date: '2025-01-02',
        description: '편의점',
        debitAccount: '식비',
        debitAmount: 5000,
        creditAccount: '현금',
        creditAmount: 5000
    }
];

localStorage.setItem('doubleEntryTransactions', JSON.stringify(testTransactions));
location.reload();
```
